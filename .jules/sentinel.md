## 2025-05-15 - Incomplete Webhook Implementations
**Vulnerability:** The WhatsApp webhook verification (GET) was completely open, returning the challenge without validating the verify token. Additionally, the Meta Marketing webhook (POST) lacked HMAC signature verification, and both handlers lacked request body size limits.
**Learning:** Webhook endpoints were implemented for functionality first, leaving critical security gaps. Standard Go string comparison was used for secrets, risking timing attacks, and the lack of body limits posed a DoS risk.
**Prevention:** All webhook implementations must include: 1) Constant-time token validation for GET challenges, 2) HMAC signature verification for POST payloads using `subtle.ConstantTimeCompare` or `hmac.Equal`, and 3) Request body size limits via `http.MaxBytesReader`.
