1. System Overview

The GST Invoice Manager is a comprehensive, production-ready web application designed for e-commerce automation. It bridges the gap between sales platforms (Shopify) and customer engagement (WhatsApp) while ensuring full GST tax compliance through automated reporting and invoice generation.

2. Directory Structure

  • /frontend/: React SPA powered by Vite. Contains UI for dashboards, order management, and automation settings.
  • /backend/: Go REST API.
    • cmd/: Entry points (main application, seeders).
    • internal/handler/: HTTP transport layer and request parsing.
    • internal/service/: Core business logic (the engine of the app).
    • internal/repository/: Database abstraction using GORM.
    • internal/entity/: Database models and structural definitions.
  • /monitoring/: Full observability stack configurations (Prometheus, Loki, Tempo, Grafana).
  • /nginx/: High-performance reverse proxy and SSL termination.

3. Database Schema

Managed via migration files in backend/internal/database/migrations/. Core tables include:

  • Orders & Items: orders, order_line_items (Stores transaction and tax data).
  • Customers: customers (Aggregated buyer profiles).
  • Automation: whatsapp_templates, whatsapp_triggers, whatsapp_messages_index.
  • System: users (Auth/RBAC), app_configs (Encrypted API keys/secrets).

4. Core Workflows

A. Order Ingestion

Real-time: Shopify sends webhooks to /api/webhooks/shopify. The system verifies the HMAC signature, logs the raw event in webhook_events, and asynchronously processes it into the main order tables.

Bulk Sync: Admins can trigger a historical fetch via the Shopify REST/GraphQL API to bring in past data.

B. WhatsApp Automation

An event-driven engine listens for order status updates (e.g., Paid, Fulfilled, Refunded). If a status matches a pre-defined Trigger, the system selects the associated Template, hydrates it with customer data, and dispatches it via the Meta Cloud API.

C. GST Reporting

Generates complex tax summaries including:

  • State-wise: Splitting IGST vs CGST/SGST based on shipping destination.
  • HSN-wise: Aggregating sales by Harmonized System codes for tax filing.
  • Document Summary: Tracking invoice number sequences and total counts.

5. API Routing Summary

The backend routes are defined in backend/internal/server/router.go:

  • /api/auth/login: JWT Session management.
  • /api/orders: Search, filter, and view specific order details.
  • /api/reports/...: Endpoints for summary, state-wise, and HSN reports.
  • /api/automation/whatsapp/...: Full CRUD for templates and triggers.
  • /api/configs/reveal: Admin-only endpoint to view sensitive keys.

6. Quick Start

# 1. Start Infrastructure
docker-compose -f docker-compose.monitoring.yml up -d

# 2. Start Backend
cd backend && go run cmd/main.go

# 3. Start Frontend
cd frontend && npm install && npm run dev
